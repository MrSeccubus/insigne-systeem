{# ── Social link previews (#140) ─────────────────────────────────────── WhatsApp/Slack/Facebook/Twitter render a card from these tags. The favicon is an SVG, which most scrapers (notably WhatsApp) won't render, so og:image points at an absolute-URL raster (the 512×512 PNG icon). URLs must be absolute — scrapers don't resolve root-relative paths — hence base_url (config.base_url, the canonical public URL). #} {# ── PWA (#101) ───────────────────────────────────────────────────── #} {# ── Offline read-only mode (#101) ───────────────────────────────────── Toggle ``body.offline`` (CSS greys out edit controls and reveals the banner). We set offline on load (``navigator.onLine`` is unreliable — stays true on reload under throttling / no internet — so we confirm with one ``/ping`` probe, which the SW never intercepts) and on the ``offline`` event. We deliberately do NOT auto-clear: a single ping getting through doesn't mean the app loads reliably on a spotty link, and the page is still showing stale cached data. Reconnecting is the user's choice via the banner's "Opnieuw proberen" link, which reloads (re-tests the app and fetches fresh data). Offline is strictly read-only, so we cancel every HTMX request: edit POSTs and live partials (e.g. the sign-off counter) would only fire doomed requests. Full-page navigation still works — the browser/service-worker handles that, not HTMX. #}
