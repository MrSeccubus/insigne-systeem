{# ── PWA (#101) ───────────────────────────────────────────────────── #} {# ── Offline read-only mode (#101) ───────────────────────────────────── Toggle ``body.offline`` (CSS greys out edit controls and reveals the banner). ``navigator.onLine`` reacts instantly to connectivity events but is unreliable: it reports the machine's link state, not whether requests actually reach the server, so it stays ``true`` on reload under throttling and on "connected but no internet". So we also probe ``/ping`` (which the service worker never intercepts) — a failed fetch means truly offline. Offline is strictly read-only, so we cancel every HTMX request: edit POSTs and live partials (e.g. the sign-off counter) would only fire doomed requests. Full-page navigation still works — that's the browser/service-worker, not HTMX. #}
